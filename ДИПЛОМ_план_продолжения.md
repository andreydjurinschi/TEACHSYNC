### II. ПРОЕКТИРОВАНИЕ МИКРОСЕРВИСНОЙ СИСТЕМЫ TEACHSYNC

#### 2.1 Функциональные и нефункциональные требования

Функциональные требования:

- авторизация пользователей;
- управление пользователями и ролями;
- управление курсами, группами и темами;
- назначение преподавателей на курсы;
- создание и изменение расписания;
- проверка конфликтов по времени, аудитории, группе и преподавателю;
- создание запросов на замену;
- подтверждение, отклонение и отмена замен;
- сохранение уведомлений;
- real-time отправка уведомлений активным пользователям;
- настройка пользовательских предпочтений уведомлений.

Нефункциональные требования:

- модульность;
- масштабируемость;
- отказоустойчивость на уровне событий;
- защищенный доступ через JWT;
- разграничение прав по ролям;
- удобство веб-интерфейса;
- возможность расширения новыми типами событий.

#### 2.2 Общая архитектура системы

Нужно вставить компонентную диаграмму.

Компоненты:

- Angular frontend;
- Gateway-service;
- Auth-service;
- Users-service;
- Course-service;
- Schedule-service;
- Replacement-service;
- Notification-service;
- Kafka;
- отдельные базы данных сервисов.

Ключевая мысль: каждый сервис отвечает за свою предметную область, а события позволяют сервисам обмениваться изменениями без жесткой связности.

#### 2.3 Описание микросервисов

Auth-service:

- аутентификация;
- выдача JWT;
- получение информации об аккаунте.

Users-service:

- хранение пользователей;
- роли `ADMIN`, `MANAGER`, `TEACHER`;
- специализации преподавателей;
- внутренние API для других сервисов.

Course-service:

- курсы, группы, категории, темы;
- связь курс-группа;
- назначение преподавателя;
- Kafka-события по назначению преподавателей.

Schedule-service:

- аудитории;
- расписание;
- проверка конфликтов;
- поиск доступных преподавателей;
- изменение расписания;
- Kafka-событие `SCHEDULE_UPDATED`.

Replacement-service:

- запросы замен;
- подтверждение, отклонение, отмена;
- работа с доступными преподавателями;
- Kafka-события по заменам.

Notification-service:

- хранение уведомлений;
- unread count;
- mark as read;
- настройки уведомлений;
- Kafka consumers;
- real-time push через SSE.

Gateway-service:

- единая точка входа;
- маршрутизация запросов к сервисам.

Современные образовательные учреждения используют различные программные и вспомогательные решения, предназначенные для организации учебного процесса.
